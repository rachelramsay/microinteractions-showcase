Forms & inputs

The most practical microinteractions — they live on every signup, checkout, and settings page. Type into each field to see the feedback. Plain CSS/JS on the left, GSAP on the right.

Plain CSS / vanilla JS GSAP

1 · Floating label

The placeholder lifts into a label as you focus or type.

Plain · CSS only

Pure CSS using :placeholder-shown — no JS at all.

GSAP

GSAP eases the label with a spring; easy to add overshoot or colour stops.

Plain — CSS

<!-- placeholder=" " is the trick -->
input:focus + label,
input:not(:placeholder-shown) + label {
  transform: translateY(-9px) scale(.78);
  color: var(--accent);
}

GSAP — JS

const up = { y:-9, scale:.78, color:'#6c8cff' };
input.addEventListener('focus', () =>
  gsap.to(label, { ...up, ease:'back.out(2)' }));
input.addEventListener('blur', () => {
  if (!input.value) gsap.to(label, { y:0, scale:1 });
});

2 · Inline validation

Validate an email as the user types — confirm or warn instantly.

Plain
✓✕

Toggles a valid/invalid class; CSS colours the border & icon.

GSAP
✓✕

Same logic, plus a quick shake on invalid — a tactile “try again”.

Plain — JS

const ok = /^[^@\s]+@[^@\s]+\.[^@\s]+$/.test(v);
field.classList.toggle('valid', ok);
field.classList.toggle('invalid', !ok && v);

GSAP — JS

// same valid/invalid logic, plus:
if (!ok && v)
  gsap.fromTo(field, { x:-6 },
    { x:0, duration:.4, ease:'elastic.out(1,0.3)' });

3 · Password strength meter

A bar fills and shifts colour as the password gets stronger.

Plain · CSS transition

JS sets width & colour; a CSS transition smooths the change.

GSAP

GSAP tweens the width with eased motion and animates the colour through the spectrum.

Plain — JS + CSS

fill.style.width = (score/4*100) + '%';
fill.style.background = colors[score];

/* CSS */
.pw-fill { transition: width .3s, background .3s; }

GSAP — JS

gsap.to(fill, {
  width: (score/4*100) + '%',
  backgroundColor: colors[score],
  duration: .4, ease: 'power2.out'
});

4 · Character counter

A live count that warns as you approach the limit (140 chars).

Plain
0 / 140

Count updates on input; a CSS class recolours near the limit.

GSAP
0 / 140

GSAP pops the counter with a scale punch each time you cross into the danger zone.

Plain — JS

count.textContent = len + ' / 140';
count.classList.toggle('warn', len > 120);
count.classList.toggle('over', len > 140);

GSAP — JS

// same recolour, plus a punch on entering warn zone:
if (len > 120 && prev <= 120)
  gsap.fromTo(count, { scale:1.5 },
    { scale:1, ease:'back.out(4)' });
