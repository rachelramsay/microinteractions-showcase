1 · Hover lift & tilt

The card responds to the pointer. Move your mouse over each.

Plain · CSS only
Lift on hover

CSS can lift & scale on :hover — but it can’t follow the cursor.

GSAP
3D tilt

GSAP reads the pointer position and tilts the card in 3D toward it.

Plain — CSS

.tilt:hover {
  transform: translateY(-7px) scale(1.03);
  box-shadow: 0 20px 38px -14px rgba(108,140,255,.7);
}
// pointer-tracking tilt isn't possible in CSS

GSAP — JS

card.addEventListener('mousemove', (e) => {
  const r = card.getBoundingClientRect();
  const px = (e.clientX - r.left) / r.width - .5;
  const py = (e.clientY - r.top) / r.height - .5;
  gsap.to(card, { rotationY: px * 22,
    rotationX: -py * 22, duration: .4 });
});

Plain — CSS

.rm-extra { display: grid;
  grid-template-rows: 0fr;
  transition: grid-template-rows .35s; }
.rm.open .rm-extra { grid-template-rows: 1fr; }

GSAP — JS

const open = card.classList.toggle('open');
gsap.to(extra, { height: open ? 'auto' : 0,
  opacity: open ? 1 : 0, duration: .35,
  ease: 'power2.out' });

GSAP tweens rotationY with easing — add a scale dip for a “lift” feel.

Plain — CSS

.flip { transform-style: preserve-3d;
  transition: transform .55s; }
.flip.flipped { transform: rotateY(180deg); }
.back { transform: rotateY(180deg);
  backface-visibility: hidden; }

GSAP — JS

flipped = !flipped;
gsap.to(card, { rotationY: flipped ? 180 : 0,
  duration: .6, ease: 'power2.inOut' });
// card has transformStyle: preserve-3d
