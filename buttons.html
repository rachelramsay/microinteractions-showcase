Category 01

Buttons & clicks

The bread-and-butter interactions. Each is built with plain CSS/JS on the left and GSAP on the right — hover, click, and tap to feel the difference, then hit View code to compare the source.

Plain CSS / vanilla JS GSAP

1 · Hover & press feedback

Lift on hover, depress on click.

Plain · CSS only

Pure :hover / :active with a transition. Zero JS.

GSAP

Springy, interruptible tween — hover out mid-animation and it eases back.

Plain — CSS

.btn {
  transition: transform .18s cubic-bezier(.2,.8,.2,1),
              box-shadow .18s, background .18s;
}
.btn:hover {
  transform: translateY(-3px);
  box-shadow: 0 12px 24px -8px rgba(108,140,255,.6);
}
.btn:active { transform: translateY(0) scale(.97); }

GSAP — JS

btn.addEventListener('mouseenter', () =>
  gsap.to(btn, { y: -4, scale: 1.04,
    duration: .35, ease: 'back.out(2)' }));

btn.addEventListener('mouseleave', () =>
  gsap.to(btn, { y: 0, scale: 1,
    duration: .4, ease: 'power3.out' }));

2 · Click ripple

A ripple emanates from the exact click point (Material style).

Plain · CSS keyframes

JS places a span at the click point; CSS keyframes scale & fade it.

GSAP

GSAP animates scale & opacity with eased control, then self-cleans.

Plain — JS + CSS

// place span at click, then:
span.classList.add('ripple');

/* CSS */
.ripple { animation: ripple .6s ease-out forwards; }
@keyframes ripple { to {
  transform: scale(2.6); opacity: 0; } }

GSAP — JS

gsap.fromTo(span,
  { scale: 0, opacity: .55 },
  { scale: 2.6, opacity: 0,
    duration: .7, ease: 'power2.out',
    onComplete: () => span.remove() });

3 · Async loading state

Idle → spinner → success checkmark.

Plain · CSS spin

Spinner rotates via CSS; JS swaps label and toggles a success class.

GSAP

A timeline cross-fades label ↔ spinner ↔ check — phases transition, not snap.

Plain — JS

label.innerHTML = '<span class=spinner></span>';
setTimeout(() => {
  btn.classList.add('is-success');
  label.textContent = '✓ Saved';
}, 1500);

GSAP — timeline

gsap.timeline()
  .to(label, { y:-8, opacity:0, onComplete: showSpinner })
  .to(label, { y:0, opacity:1 })
  .to({}, { duration: 1.6 })   // "work"
  .to(label, { opacity:0, onComplete: showCheck })
  .to(label, { scale:1, ease:'back.out(2.5)' });

4 · Copy to clipboard

Confirm success with a transient message.

Plain

Text appears, then a CSS opacity transition fades it out.

GSAP

GSAP pops “Copied!” with a bounce, holds, then slides it away.

Plain — JS

navigator.clipboard.writeText(text);
msg.textContent = '✓ Copied!';
msg.style.opacity = 1;
setTimeout(() => msg.style.opacity = 0, 1300);

GSAP — JS

navigator.clipboard.writeText(text);
gsap.fromTo(msg, { y:10, opacity:0, scale:.8 },
  { y:0, opacity:1, scale:1, ease:'back.out(3)' });
gsap.to(msg, { y:-10, opacity:0, delay:1.2 });

5 · Like / heart burst

A satisfying toggle — the moment of delight.

Plain · CSS pop

CSS handles the colour change and a keyframe “pop”.

GSAP

GSAP springs the icon and fires a particle burst — hard in CSS alone.

Plain — CSS

.heart.is-liked path { fill: #ff6b81; }
.heart.is-liked svg { animation: pop .4s; }
@keyframes pop {
  40% { transform: scale(1.35); } }
// no particle burst without more work

GSAP — JS

gsap.fromTo(svg, { scale:1 },
  { scale:1.4, yoyo:true, repeat:1 });

for (let i=0; i<10; i++) {           // burst
  gsap.to(particle, { x: cos(a)*38, y: sin(a)*38,
    scale:0, opacity:0, onComplete: remove });
}
